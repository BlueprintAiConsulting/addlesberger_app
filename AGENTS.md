# Travellir Management — Agent Instructions

> **READ THIS BEFORE MAKING ANY CHANGES.**

## Architecture Overview

This repo serves **TWO separate applications** from different directories:

### 1. PRM (Property Rental Manager) — `index.html` (root)
- **Served via:** GitHub Pages from the root of this repo
- **URL:** GitHub Pages URL (BlueprintAiConsulting/travellir-management)
- **Entry point:** `/index.html` (root — ~11,000+ lines)
- **External scripts loaded at bottom of index.html:**
  - `property-intake.js` — Property onboarding & Hostfully push
  - `bookings.js` — Live Hostfully booking sync
  - `ai-concierge-ui.js` — Owner AI concierge widget
- **Deployment:** Automatic via GitHub Pages on push to `main`
- **This is the admin dashboard used daily by the Travellir team.**

### 2. Public Booking Site — `site/` directory
- **Served via:** Firebase Hosting (`firebase.json` → `"public": "site"`)
- **URL:** travellir-booking.web.app / travellir.com
- **Entry point:** `site/index.html`
- **Has its own separate copies** of some JS files (e.g., `site/property-intake.js`, `site/ai-concierge-ui.js`)
- **Deployment:** `firebase deploy --only hosting`
- **This is the guest-facing booking website.**

### ⚠️ `site/travellir-management-index.html`
This is an **older, outdated copy** of the PRM (~2,767 lines). It is NOT the live PRM. Do NOT edit this file thinking it will fix the PRM dashboard. The live PRM is **always** the root `index.html`.

## 🚨 CRITICAL: NEVER OVERWRITE `site/index.html` WITH THE PRM DASHBOARD

**This has caused a production outage before.** The PRM dashboard (`index.html` at root, ~850KB) was accidentally placed as `site/index.html`, which gets deployed to `travellir-booking.web.app` / `travellir.com` via Firebase Hosting. Every visitor saw a login screen instead of the public booking site.

**Rules to prevent this:**
- `site/index.html` must ALWAYS be the **public booking site** (~64KB, title: "Vacation Rentals in West Virginia, Florida & Scottsdale")
- `site/index.html` must NEVER contain `PRM Dashboard`, `Property Management`, or `#loginScreen`
- The root `index.html` (PRM, ~850KB) must NEVER be copied, moved, or written into `site/index.html`
- **Before any Firebase deploy**, run `./deploy.sh` which has automated safety checks
- If you are editing or regenerating `site/index.html`, verify the title tag says "Vacation Rentals", NOT "Property Management"

**How to tell them apart:**
| File | Size | Title | Purpose |
|------|------|-------|---------|
| `./index.html` | ~850KB | "Travellir — Property Management" | PRM dashboard (GitHub Pages) |
| `./site/index.html` | ~64KB | "Travellir — Vacation Rentals..." | Public booking site (Firebase) |

## Critical Rules

1. **NEVER overwrite `site/index.html` with the PRM dashboard.** See warning above. The deploy script will block this, but don't even try.

2. **NEVER delete the closing tags at the bottom of `index.html`:**
   ```html
   </script>
   <script src="property-intake.js?v=2.4"></script>
   <script src="bookings.js?v=1.0"></script>
   <script src="ai-concierge-ui.js?v=1.0"></script>
   </body>
   </html>
   ```
   Removing these will break the ENTIRE PRM — every screen goes blank.

3. **Always commit AND push to `main`** after making changes. The PRM is deployed via GitHub Pages — local changes mean nothing until pushed.

4. **Firebase deploy (`firebase deploy --only hosting`) does NOT deploy the PRM.** It deploys the public booking site from `site/`. These are completely separate. Always use `./deploy.sh` instead of raw `firebase deploy`.

5. **Do NOT add `enablePersistence()` to the Firebase init without thorough testing.** A previous attempt broke the app by blocking the initial render.

6. **The root `index.html` is a monolithic SPA.** All PRM features (dashboard, properties, calendar, guest comms, owner portal, cleaning, channels, etc.) live inline in this single file. External scripts (`property-intake.js`, `bookings.js`, `ai-concierge-ui.js`) are loaded at the bottom.

6. **Hostfully is the source of truth.** All property data flows through Hostfully. The PRM reads from and writes to Hostfully via API. Any changes pushed to Hostfully propagate to Airbnb, VRBO, etc.

## Tech Stack

- **Frontend:** Vanilla JS, no framework, no build step
- **Backend:** Firebase (Firestore, Auth, Cloud Functions, Storage)
- **Hosting:** GitHub Pages (PRM) + Firebase Hosting (booking site)
- **APIs:** Hostfully API, Google Gemini AI, Google Maps
- **No package manager needed for frontend** — all dependencies loaded via CDN

## Testing Changes

After editing `index.html` or any root-level JS file:
1. Verify locally by opening `index.html` in a browser
2. `git add . && git commit -m "description" && git push origin main`
3. Wait 30-60 seconds for GitHub Pages to rebuild
4. Hard refresh (Cmd+Shift+R) the PRM URL to verify

## ⛔ NEVER Delete Existing Code During "Audits" or "Refactors"

A previous "audit" commit deleted 3,891 lines of working code including critical functions (`getDb`, `GROUP_LIVING_PROPS`, `PROPERTIES`, owner portal code, calendar code, etc.) which caused the entire PRM to show blank screens.

**Rules for code audits/refactors:**
- **NEVER delete a function or variable** unless you have confirmed zero references to it in the entire codebase
- **NEVER "move code to Firestore"** without implementing the Firestore read that replaces it
- **Always run `grep -c "FUNCTION_NAME" index.html`** before deleting anything
- **Test in a browser** after every change — if any screen is blank, you broke something

## Essential Globals (NEVER DELETE)

These variables and functions are referenced throughout the entire PRM. Deleting any one of them will crash the app:

| Name | Purpose |
|------|---------|
| `PROPERTIES` | Array of all STR property data |
| `GROUP_LIVING_PROPS` | Array of all group living property data |
| `OWNER_DATA` | Array of all owner accounts |
| `getDb(callback)` | Core Firestore accessor — used 25+ times |
| `loadFirebaseSDKs(callback)` | Firebase SDK lazy loader |
| `renderPage()` | Main page router — renders all screens |
| `nav(page, el)` | Navigation handler |
| `showToast(msg)` | UI notification system |
| `_sharedDb` / `crDb` | Firestore instance references |
| `isDemoMode` | Demo/live mode flag |
